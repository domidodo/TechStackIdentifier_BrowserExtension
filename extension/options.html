TechStack Identifier Configuration

1. Required Meta-Tag

To detect the environment, the website must include this tag in the <head>:

<meta name="company:tech" content="S" />

(Content values like "S", "P", ... are matched against your profiles below.)

JSON Configuration

Documentation

profiles

Define reusable display sets. Each key (e.g. "S" = Staging) needs a label, color, and badge text.

domainMapping

Maps hostnames to profiles. Includes all subdomains (e.g. google.com also covers mail.google.com).

exceptions

Forces display for EXACT URL matches only. This bypasses the meta-tag check completely.

Example Template

{
  "profiles": {
    "standard": {
      "S": { "label": "Staging", "color": "#1E90FF", "badge": "STG" },
      "P": { "label": "Production", "color": "#FF0000", "badge": "PRD" }
    }
  },
  "domainMapping": {
    "default": "standard",
    "example.com": "standard"
  },
  "exceptions": {
    "https://example.com/special": { 
      "label": "Exception", "color": "#000", "badge": "EX" 
    }
  }
}
